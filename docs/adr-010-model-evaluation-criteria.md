# ADR-010: モデル評価基準と学習継続・見切りの判断指針

## 1. ステータス
**承認済み**

## 2. コンテキスト
現在、複数のTTSモデル（XTTS, GPT-SoVITS, Fish-Speech, StyleTTS2）を並行して検証している。リソースを効率的に配分し、志ん生音声の再現度を最大化するため、各モデルの「伸び代」を客観的に評価し、学習を継続するか、あるいは「見限る」かの基準を明確にする必要がある。

## 3. 意思決定事項

評価スコア（0-100）に基づき、以下の3段階でモデルの進退を判断する。

### A. 期待大 (学習継続)
- **基準:** 平均スコア 75以上
- **アクション:** 
  - 追加のファインチューニング（LoRA等）を優先的に実施。
  - データセットの質をさらに高め、微細なニュアンスの再現に注力する。

### B. 見極め中 (条件付き継続)
- **基準:** 平均スコア 60 - 74
- **アクション:**
  - 特定の欠点（滑舌、ノイズ、イントネーションの不自然さ）が改善可能か分析。
  - LoRAによる改善が見込める場合は、1スプリント限定で追加学習を実施。改善が見られない場合は「見切り」に移行。

### C. 見切り検討 (構造的見直し)
- **基準:** 平均スコア 60未満
- **アクション:**
  - 現状のアーキテクチャでは志ん生の声質（枯れ声、独特の間）を再現できないと判断。
  - 抜本的なパラメータ変更、あるいは当該モデルの使用を停止し、他のモデルにリソースを集中させる。

## 4. 評価統計の自動化
`src/eval-batch-stats.py` を使用して、`tts_outputs` ディレクトリ内の全ファイルを一括評価し、モデルごとの統計レポートを自動生成する。このレポートを週次の「見極め会議」の基礎資料とする。

## 5. 結果
- リソースの集中投下が可能になり、開発速度が向上する。
- 主観的な「なんとなく」ではなく、数値に基づいたモデル選定が可能になる。
- 失敗の早い段階での検知（Fail Fast）が可能になる。
