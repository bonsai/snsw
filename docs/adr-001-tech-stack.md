# ADR-001: 五代目古今亭志ん生の音声再現における技術スタックの選定

## 1. ステータス
**承認済み**（2026年現在の推奨技術に基づく）

## 2. コンテキスト
五代目古今亭志ん生特有の「枯れた声」や落語特有の「間」をAIで高品質に再現することを目指します。当初候補に挙がっていたTortoise TTSは、2026年時点ではメンテナンスが停止しており、生成速度や実用性の面で課題があります。そのため、現在の技術トレンドに合わせた最適なモデルとワークフローの選定が必要です。

## 3. 意思決定
以下の技術スタックを採用することを決定しました。

### **メインモデル: Coqui XTTS-v2**
*   **理由:** 2026年時点で最も日本語品質が高く、数秒の参照音声でクローニングが可能な「ゼロショット」性能に優れているため。
*   **運用:** 1時間〜10時間程度の高品質な音声データを用いた**ファインチューニング (FT)** を実施し、志ん生特有のニュアンスを学習させます。

### **補助モデル: RVC (Retrieval-based Voice Conversion)**
*   **理由:** XTTS-v2で生成した音声に、さらに志ん生の「声の質感」を強く反映させるためのハイブリッド構成として採用します。
*   **用途:** 感情表現や「間」の微調整が必要な場合、変換型であるRVCを組み合わせることで最強のクオリティを狙います。

### **データ処理パイプライン**
*   **収集:** **yt-dlp** を使用し、YouTube等のアーカイブから高品質（WAV形式）で音源を抽出します。
*   **文字起こし:** **Whisper (large-v3)** を用い、学習に不可欠な正確なテキストデータを作成します。
*   **編集:** **Audacity** や **pydub** で、拍手や笑い声をカットし、純粋な話し声のみを抽出します。

### **実行・学習環境**
*   **UI:** 初心者でも扱いやすい **AllTalk TTS** または **xtts-finetune-webui** を使用します。
*   **ハードウェア:** 学習の安定性と速度を確保するため、**16GB〜24GB以上のVRAM**を搭載したGPU（NVIDIA A100等）環境を推奨します。

## 4. 決定による影響（コンシクエンス）

### **メリット**
*   **高品質な再現:** 志ん生の独特な「間」や息遣いまで再現できる可能性が高まります。
*   **柔軟性:** テキストからの音声生成（TTS）と、既存音声の変換（VC）の両方が可能です。

### **デメリット/リスク**
*   **著作権:** 1950〜60年代の音源には隣接権が絡むため、公開や商用利用には権利元（NHKやコロムビア等）の確認が必要です。
*   **計算資源:** 高品質なファインチューニングには、ハイスペックなPC環境と学習時間が要求されます。
