# ADR-011: TTSモデル選定と複数モデル共存環境の構築

## 1. ステータス
**承認済み**

## 2. コンテキスト
現在のTTS生成において「似ていない」という課題を解決するため、複数のアプローチ（LLMベース、拡散モデルベース、少データクローニング）を持つ主要な5つのモデルを統合し、柔軟に切り替え可能な環境を構築する。

## 3. 決定事項
以下の5つのTTSエンジンを同一のDocker環境内に共存させ、`src/tts-multi-selector.py` で切り替え可能にする。

### 1. XTTS v2 (Coqui TTS)
- **特徴**: 多言語対応、ゼロショット・ボイスクローニングのデファクトスタンダード。
- **長所**: 依存関係が比較的軽量、日本語を含む多言語クオリティが安定。
- **短所**: 表現力の限界があり、特定の話者の微細な特徴の再現が難しい場合がある。

### 2. Qwen2-Audio (qwntts / transformers)
- **特徴**: Alibabaが開発した7Bクラスのマルチモーダル音声理解・生成モデル。
- **長所**: 大規模モデル特有の自然な抑揚と文脈理解。
- **短所**: 7Bモデルのためメモリ消費が非常に激しい（約16GB+）。

### 3. GPT-SoVITS
- **特徴**: 1分程度の音声データで極めて高品質なクローニングが可能なモデル。
- **長所**: **日本語の再現性が圧倒的に高い**。
- **短所**: モデルの構造が複雑で、推論に特定のラッパーを必要とする。

### 4. Fish-Speech
- **特徴**: VQ-GANとLLMを組み合わせたSOTAな音声生成モデル。
- **長所**: 最新のアーキテクチャで、音質とクローニング精度のバランスが良い。
- **短所**: 依存関係が激しく、環境構築の難易度が高い。

### 5. StyleTTS2
- **特徴**: 拡散モデルのようなスタイルマッチングを使用する高速・高品質モデル。
- **長所**: 感情表現やスタイルの制御が強力。**推論が非常に高速**。
- **短所**: 日本語の事前学習済みモデルが限定的。

## 4. コンシケンス（影響）
- **ポジティブ**: 音質の不満に対し、異なるアプローチのモデルで即座に試行錯誤が可能。
- **ネガティブ**: Dockerイメージのサイズが大幅に増大する。

## 5. 今後の展望
- 各モデルでの生成結果を比較し、最も「似ている」結果が得られたモデルをメインエンジンとして採用する。
